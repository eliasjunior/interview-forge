# Algorithm Pattern Study System

## What this is

A personal learning tracker for algorithm practice. After each coding interview or exercise session, the Problem Tracker sheet is updated to capture what was learned and what to revisit.

## File

`algorithm_pattern_study_system.xlsx`

## Active sheet: Problem Tracker

Tracks one problem per row. The guiding principle is written at the top of the sheet:

> After each problem, capture the recognition pattern, the exact point of friction, the mistake to avoid, and when to test recall again.

### Columns

| Column | What to fill in |
|---|---|
| Problem | Problem name (e.g. "Merge Sorted Array") |
| Pattern | The algorithmic pattern or technique used |
| Difficulty | Easy / Medium / Hard |
| Tricky Part | The specific point of friction — what was hard or confusing |
| Mental Model | The core insight or framing that makes the solution click |
| Common Mistake | The mistake to avoid next time |
| Complexity | Time and space complexity |
| Re-solved Without Help? | Yes / No |
| Date Last Reviewed | Date of the session |
| Next Review | How many days until the next recall attempt |

## When to update

At the end of each algorithm interview session. The user will ask for this explicitly.

## Sheets to ignore for now

`Dashboard` and `Pattern Library` are not in active use.
